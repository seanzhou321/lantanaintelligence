The Methodology Gap

AI coding changes which parts of software development matter. Implementation is no longer the expensive constraint — specification quality is. Teams that import AI tools into existing Agile workflows without changing anything else discover this the hard way: invisible structural drift, accumulated technical debt, and code that looks correct but doesn't realize what was actually intended.

My consulting practice is built on three original research papers that examine this problem from three angles — methodology, architecture governance, and verification — and propose a connected framework for addressing it: the Stratified Behavioral Refinement (SBR) framework. I work directly with engineering teams to assess their current practices and introduce the discipline that makes AI generation trustworthy.
